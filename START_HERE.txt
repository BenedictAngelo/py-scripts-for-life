🔐 SECURITY INFO:

Key File Management:
- key.key is created automatically on first run
- DO NOT lose key.key - you won't be able to decrypt passwords!
- DO NOT share key.key
- DO NOT commit to version control (it's in .gitignore)
- Keep it safe in a secure location

Password Storage:
- All passwords are encrypted with Fernet
- Stored in passwords.txt (also excluded from git)
- Format: website|email|encrypted_password|timestamp
- Only decryptable with key.key

❓ FREQUENTLY ASKED QUESTIONS:

Q: How do I run the application?
A: poetry run python src/main.py

Q: What's the Password Manager?
A: A secure CLI tool to generate, store, and manage encrypted passwords.

Q: Is it safe to use?
A: Yes. Passwords are encrypted with Fernet (industry-standard).

Q: Can I lose my passwords?
A: Only if you lose key.key. Keep it safe!

Q: Can I add more modules?
A: Yes! Follow the existing pattern in src/yt-to-mp3/ (placeholder).

Q: Where's the database?
A: Uses file-based storage (passwords.txt). Database coming in future.

Q: How do I contribute?
A: Follow the patterns in existing code and document your changes.

Q: Is there a GUI?
A: Not yet. Currently CLI only. GUI possible as future enhancement.
